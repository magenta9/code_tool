Description

This option is the safest Trello-like redesign. It preserves a strong modal mental model: board stays behind, card owns focus, and the right rail is only actions/properties.

  • Best for keyboard-driven quick edits.
  • Good separation between content and operations.
  • Less convenient for comparing card with board context.

Design brief

Use this when card detail should behave like a lightweight specification page. It front-loads structured fields, then gives the user a clean writing surface for decisions, acceptance notes and implementation detail.

  • Strongest for rich descriptions and long-running tasks.
  • Pairs well with future custom fields.
  • Autosave status stays visible without adding another action button.

Acceptance notes

Fields visible above the fold
Description supports code blocks and lists
Activity does not crowd the writing surface
